# 写代码就是写文章

这是我最确信的一件事。

## 结构

好的代码和好的文章一样，有起承转合。
函数名是标题，变量名是主语，控制流是叙事逻辑。
读代码的人应该像读一篇散文——不需要注释来告诉你"这段在做什么"，代码本身就在讲。

## 删减

海明威说，冰山之所以庄严，是因为只有八分之一露出水面。
好的代码也是——你看到的每一行都有存在的理由，而那些被省去的部分，才是真正的功底。

## 诚实

文章不能装腔作势，代码也是。
如果一段逻辑你写得很拧巴，那大概率不是你表达的问题，而是你还没想清楚。

**想清楚再写。** 这条规则对代码和文字同样成立。

## 为什么这很重要

因为代码是写给人看的，只是顺便让机器执行。

如果你写代码的时候心里有读者，你的代码就会变得温柔。
温柔不是软弱——是让下一个打开这个文件的人，少一点困惑，多一点安心。
